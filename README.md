# 한국 주식 바닥 스크리너 (Korea Bottom Screener)

코스피·코스닥 전종목을 매일 자동으로 훑어 **바닥 신호**가 강한 종목을 뽑고, 그중 실제로
**턴어라운드(방향 전환)**가 확인된 종목을 따로 골라주는 도구입니다.
서버를 상시 켜둘 필요 없이 GitHub Actions가 매일 한 번 계산하고, 결과를 웹 화면으로 보여줍니다.

## 구성

```
signals.py                 # 점수 엔진 (바닥 7개 + 턴어라운드 5개 → 0~100점). 네트워크 불필요, 단독 테스트 가능
screener.py                # pykrx로 데이터 수집 → 채점·분류 → results.json 저장
requirements.txt
.github/workflows/daily.yml # 매일(평일 KST 20:00) 자동 실행
docs/index.html             # 결과 화면 (results.json을 읽어 confirmed_turnaround/watching 그룹으로 표시)
results.json               # 매 실행마다 갱신되는 결과 (Actions가 커밋)
```

## 왜 두 그룹으로 나누는가

바닥 신호 7개는 전부 "매도세가 소진됐다·역사적으로 싸다"만 본다. 하방이 막혔다는 증거일 뿐
실제로 오르기 시작했다는 증거는 아니라서, 바닥은 맞는데 몇 달~몇 년 안 오르는
"죽은 돈(dead money)"에 물릴 수 있다. 그래서 "실제로 방향을 틀었는지"만 보는 턴어라운드
신호 5개를 완전히 별도로 채점하고, 두 점수의 교집합을 최종적으로 강조한다.

## 바닥 신호 7개 (가중평균, 각 0~100점) — 종합 60점 이상만 통과

가중치(`signals.py`의 `BOTTOM_WEIGHTS`)는 **백테스트로 검증된 값이 아니라, 신호가 얼마나
"직접적인 증거"인지에 대한 초기 추정값**입니다. 나중에 과거 바닥 사례로 역검증해 조정할
예정입니다. `composite_score(scores)`를 weights 없이 호출하면 예전처럼 동일가중으로도
계산할 수 있어 두 방식을 백테스트로 비교 가능합니다.

1. **기관·외국인 매집** (25) — 20일 누적 순매수 ÷ 유통시총, 주가가 조용할수록 가중
2. **거래량 고갈** (18) — 6~25일 전(최근 5일 제외) 거래량 중앙값 ÷ 과거 120일 중앙값 (낮을수록 고점).
   최근 5일을 빼는 이유는 턴어라운드 신호 '거래량 동반 상승'과 구간이 겹치지 않게 하기 위함 —
   그래야 "오래 조용했다 → 최근에 반응했다" 패턴이 서로 다른 날짜로 깔끔하게 갈린다.
3. **PBR 역사적 저점** (15) — 5년 PBR 밴드 내 백분위 (낮을수록 고점)
4. **공매도 감소** (13) — 현재 공매도잔고비중 ÷ 3개월 최고 (낮을수록 고점)
5. **상대강도** (13) — 60일 수익률의 **업종지수** 대비 초과분 (업종지수보다 버틸수록 고점).
   코스피 전체와 비교하면 "반도체 사이클 하락기에 반도체 종목이 코스피보다 선방"한 게
   업종 전체가 덜 빠진 것뿐인지 종목 고유의 강도인지 구분이 안 되는 섹터 편향이 생겨서,
   KRX 업종지수(코스피 24개·코스닥 22개, `get_index_ticker_name`으로 이름에 시장명이
   없는 것만 필터링해 사이즈/스타일 지수를 제외)로 바꿨다. 업종 매핑이 안 되는 종목은
   코스피 전체로 폴백.
6. **변동성 수축(밴드 스퀴즈)** (11) — 20일 볼린저밴드폭이 자체 과거 히스토리 대비 얼마나 좁은지
7. **배당수익률(조건부)** (5) — 배당주 한정, 배당성향 80%↑·삭감이력은 함정으로 배제

> **액면분할·무상증자 방어**: pykrx가 쓰는 전종목 스냅샷 함수엔 수정주가 옵션이 없어
> 원주가 그대로 나온다. `collect_ohlcv_matrix`가 하루 등락 ±30%(KRX 상하한가) 밖인
> 날은 그 종목만 그 날짜에서 걸러내고, 그래도 남는 잔존 이상치(예: 여러 날에 걸쳐
> 반영된 경우)는 채점 직전에 `has_unadjusted_split_jump`로 한 번 더 확인해
> 최근 60일 내 전일 대비 2배↑/0.5배↓ 지점이 있으면 상대강도·이평선돌파·단기고점돌파
> 3개 신호만 그 종목에 한해 제외(None)한다.

## 턴어라운드 신호 5개 (가중평균, 별도 종합점수) — 2개 이상 50점↑이면 confirmed_turnaround

가중치는 `signals.py`의 `TURNAROUND_WEIGHTS`. 마찬가지로 초기 추정값입니다.

1. **매집 가속** (30) — 최근 5일 일평균 순매수 ÷ 이전 15일 일평균 순매수
2. **상대강도 가속** (25) — 최근 10일 상대강도 - 이전 10일 상대강도 (개선 속도). 상대강도(⑤)와
   동일하게 업종지수 대비(폴백은 코스피 전체).
3. **거래량 동반 상승** (20) — 최근 5일 평균거래량 ÷ 20일 평균거래량 (높을수록 고점).
   바닥 신호 '거래량 고갈'과 정반대 방향이니 혼동 주의.
4. **이평선 돌파** (15) — 종가가 60일선 위, 20일선이 60일선 위(정배열)
5. **단기 고점 돌파** (10) — 현재가가 최근 60일 박스권 고점 대비 몇 % 위치인지. 52주 신고가나
   역사적 전고점이 아니라 반드시 60일 기준 — PBR 역사적 저점 같은 장기 바닥 신호와
   논리적으로 충돌하지 않도록 "최근 두 달 단기 흐름 전환"만 포착

> 신용잔고·내부자/자사주 매수는 KRX 공식/비공식 API, 네이버금융 어디에도 무료로 종목별
> 조회할 방법을 찾지 못해 빠져 있습니다. (재무건전성은 DART Open API 연동으로 가능할 수 있음)

## 설치·배포 (처음 하는 사람 기준)

1. **이 폴더를 새 GitHub 저장소에 올린다.** (Claude Code로 하면 자동)
2. 저장소 **Settings → Actions → General**에서 workflow 쓰기 권한 허용.
3. **KRX 로그인 계정 준비 (2025-12-27부터 필수, 무료)**: [data.krx.co.kr](https://data.krx.co.kr)에서
   네이버/카카오로 간편가입 후, 저장소 **Settings → Secrets and variables → Actions**에서
   `KRX_ID`, `KRX_PW` 시크릿을 등록. (없으면 모든 조회가 실패합니다)
4. **Actions 탭 → daily-screener → Run workflow**로 첫 실행. (몇 분 소요)
   - 첫 실행에서 휴장일·결측 등으로 조정이 필요할 수 있습니다(정상).
5. 실행이 끝나면 `results.json`이 생기고 `docs/results.json`으로 복사됩니다.
6. **화면 띄우기**: 저장소 **Settings → Pages**에서 Source를 `main` 브랜치 `/docs` 폴더로 설정.
   `https://<계정>.github.io/<저장소>/` 주소가 생기면 폰 브라우저에서 열고
   "홈 화면에 추가"로 아이콘처럼 쓸 수 있습니다.

## 로컬에서 점수 로직만 확인

```bash
python signals.py      # 가짜 데이터로 바닥 7개+턴어라운드 5개 신호·종합점수 검증 (네트워크 불필요)
```
